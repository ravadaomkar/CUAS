# logs/

Persistent log files, one per script, written automatically by
`utils/logger.get_logger(name)`:

```
logs/
├── train.log            # scripts/train_yolo.py
├── evaluate.log           # scripts/evaluate.py
├── evaluate_modality.log   # scripts/evaluate_modality.py
├── tune.log                 # scripts/tune.py
└── inference_tta.log         # scripts/inference_tta.py
```

Each line is timestamped and includes the logger name, so `grep` across
files works for cross-script debugging, e.g.:

```bash
grep -h "mAP" logs/*.log | sort
```

This folder is git-ignored (see `.gitignore`) — logs are regenerated by
re-running the corresponding script, so they don't belong in version
control, but keeping them locally is what makes debugging a failed
overnight training run possible without re-running it.
