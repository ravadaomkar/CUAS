# analysis/

Bucketed failure cases from a scored evaluation run, generated by
`scripts/error_analysis.py`. The goal is to make "where does the model
struggle" a five-minute look through a folder of images instead of manually
diffing prediction files against labels.

```
analysis/
├── false_positive/     # model predicted a drone, no GT box matched (IoU < 0.5)
├── false_negative/       # a GT drone box was never matched by any prediction
└── difficult_cases/       # matched, but at low IoU (0.5-0.7) — borderline localization
```

Each image is saved with both the ground-truth box (blue) and predicted
box(es) (green) drawn on it, so you can immediately see *why* it failed —
wrong scale, occlusion, confusable background clutter (birds/aircraft),
low-PoT range shot, etc.

Generate these after any `scripts/evaluate.py` run:

```bash
python scripts/error_analysis.py \
    --weights checkpoints/best.pt \
    --data_dir data/real_reference \
    --conf 0.15 --num_per_bucket 25
```

This folder is git-ignored (regenerate any time) — see `.gitignore`.
